üìã Overview

AWS WAF has launched a new AI Activity Dashboard that provides centralized visibility into AI bot and AI agent traffic. Alongside this release, the detection coverage of AWS WAF Bot Control has been expanded to over 650+ unique bots and agents, making it one of the most comprehensive AI bot detection catalogs in the industry.

AI-powered bots and autonomous agents are rapidly changing web traffic patterns. AI search crawlers index content, RAG systems retrieve data in real-time, and autonomous agents execute multi-step tasks across APIs and web applications. Without visibility into this traffic, organizations risk infrastructure cost increases, application performance degradation, and unauthorized access to content.

Traffic from AI bots and agents passes through CloudFront, ALB, or API Gateway, is inspected by AWS WAF Bot Control, and visualized in the AI Activity Dashboard. Based on dashboard analysis, you can allow verified crawlers, rate-limit unverified agents, or block malicious bots.

üöÄ Setup Guide

Prerequisites

  • AWS account with AWS WAF enabled
  • Web ACL associated with a protected resource (CloudFront, ALB, API Gateway, etc.)
  • Appropriate IAM permissions
Step 1: Add Bot Control Managed Rule Group
Add the Bot Control managed rule group to your Web ACL in the AWS WAF console.
Bash AWS CLI - Add Bot Control Rule Group 
aws wafv2 update-web-acl \
  --name my-web-acl \
  --scope REGIONAL \
  --id <web-acl-id> \
  --lock-token <lock-token> \
  --default-action '{"Allow": {}}' \
  --rules '[
    {
      "Name": "AWS-AWSManagedRulesBotControlRuleSet",
      "Priority": 0,
      "Statement": {
        "ManagedRuleGroupStatement": {
          "VendorName": "AWS",
          "Name": "AWSManagedRulesBotControlRuleSet",
          "ManagedRuleGroupConfigs": [
            {
              "AWSManagedRulesBotControlRuleSet": {
                "InspectionLevel": "COMMON"
              }
            }
          ]
        }
      },
      "OverrideAction": {"Count": {}},
      "VisibilityConfig": {
        "SampledRequestsEnabled": true,
        "CloudWatchMetricsEnabled": true,
        "MetricName": "BotControlMetric"
      }
    }
  ]' \
  --visibility-config '{
    "SampledRequestsEnabled": true,
    "CloudWatchMetricsEnabled": true,
    "MetricName": "myWebACL"
  }'
Start in Count mode first to observe the impact on production traffic before enabling enforcement actions.
Step 2: View the AI Activity Dashboard
  1. Navigate to the AWS WAF Console
  2. Select the protected resource or Web ACL
  3. Open the dashboard and navigate to the Bots section
  4. Review AI traffic trends, active bot listings, and request volume analysis
Step 3: Configure Bot Control Rule Actions
Based on dashboard analysis, configure actions per bot category.
JSON Custom Rule - Block Unverified AI Bots 
{
  "Name": "BlockUnverifiedAIBots",
  "Priority": 1,
  "Statement": {
    "AndStatement": {
      "Statements": [
        {
          "LabelMatchStatement": {
            "Scope": "LABEL",
            "Key": "awswaf:managed:aws:bot-control:bot:category:ai"
          }
        },
        {
          "NotStatement": {
            "Statement": {
              "LabelMatchStatement": {
                "Scope": "LABEL",
                "Key": "awswaf:managed:aws:bot-control:bot:verified"
              }
            }
          }
        }
      ]
    }
  },
  "Action": {"Block": {}},
  "VisibilityConfig": {
    "SampledRequestsEnabled": true,
    "CloudWatchMetricsEnabled": true,
    "MetricName": "BlockUnverifiedAIBots"
  }
}

This custom rule blocks AI bots that are not verified, while allowing verified crawlers to continue accessing your resources.

‚ö†Ô∏è Limitations and Considerations

Limitations

  • Full Bot Control features require a Bot Control managed rule group subscription (additional charges apply)
  • Targeted Inspection level incurs higher charges than Common Inspection level
  • AWS WAF enforcement is required for malicious bots that do not respect robots.txt

Considerations

  • Add the Bot Control rule group in Count mode first; verify impact on legitimate traffic before switching to block actions
  • Even verified bots may send high-volume traffic; consider combining with rate-limit rules
  • Web Bot Authentication requires explicitly selecting AWSManagedRulesBotControlRuleSet Version 4.0 or later
  • AI bot traffic patterns evolve rapidly; regularly review the dashboard and adjust rules accordingly

üí° Use Cases

1Content Protection from AI Scraping
Scenario: A media company wants to prevent its proprietary content from being used for unauthorized AI model training.
Implementation:
  1. Add the Bot Control managed rule group at Common Inspection level
  2. Monitor LLM Training Crawlers category traffic in the AI Activity Dashboard
  3. Create a custom rule to block unverified LLM training crawlers
Effect: Protect intellectual property while maintaining indexing by verified AI search crawlers for search engine visibility.
2AI Search Crawler Management
Scenario: An e-commerce site wants to allow AI search crawler indexing while preventing performance degradation from excessive crawling.
Implementation:
  1. Analyze request volume per crawler in the AI Activity Dashboard
  2. Allow verified AI search crawlers
  3. Apply rate-limit rules to crawlers with excessive request volumes
Effect: Maintain inbound traffic from AI search engines while managing infrastructure load appropriately.
3Rate Limiting Autonomous AI Agents
Scenario: A SaaS provider offers services via API and wants to control high-volume API requests from autonomous AI agents.
Implementation:
  1. Add the Bot Control managed rule group at Targeted Inspection level
  2. Analyze AI agent access patterns in the AI Activity Dashboard
  3. Set rate limits on unverified agents; grant full access only to authenticated API clients
Effect: Maintain legitimate API usage while preventing excessive resource consumption by autonomous agents.

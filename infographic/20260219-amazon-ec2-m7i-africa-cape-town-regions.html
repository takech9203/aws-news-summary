Amazon EC2 M7i Instances Africa (Cape Town)

Amazon EC2 M7i インスタンスが
アフリカ (ケープタウン) リージョンで利用可能に

2026 年 2 月 19 日

☁️ 概要

Amazon EC2 M7i インスタンスがアフリカ (ケープタウン) リージョンで利用可能になりました。M7i インスタンスは AWS 専用のカスタム第 4 世代 Intel Xeon Scalable プロセッサ (Sapphire Rapids) を搭載した汎用インスタンスです。同等の x86 ベース Intel プロセッサと比較して最大 15% 優れたパフォーマンスを提供し、M6i インスタンスと比較して最大 15% 優れた価格パフォーマンスを実現します。

M7i インスタンスはゲームサーバー、CPU ベースの機械学習 (ML)、動画ストリーミングなどのワークロードに最適です。最大 48xlarge サイズまで 11 サイズを提供し、2 つのベアメタルサイズ (metal-24xl、metal-48xl) も利用可能です。さらに、4 つの内蔵 Intel アクセラレータ (DSA、IAA、QAT、AMX) を搭載しています。

❌ Before - アップデート前の課題
  • M7i インスタンスがアフリカ (ケープタウン) リージョンで利用できなかった
  • アフリカリージョンのユーザーは最新世代の Intel ベース汎用インスタンスの恩恵を受けられなかった
  • データレジデンシー要件のあるワークロードで、ケープタウンリージョンに最新の汎用インスタンスをデプロイできなかった
✅ After - アップデート後の改善
  • アフリカ (ケープタウン) リージョンで M7i インスタンスを直接起動できるようになった
  • アフリカリージョンのユーザーが AWS 専用カスタム Intel プロセッサの性能を活用できるようになった
  • データローカリティ要件を満たしながら、高性能な汎用コンピューティングを利用可能になった

M7i インスタンスは large から 48xlarge までのサイズと 2 つのベアメタルサイズを提供し、DDR5 メモリと高帯域幅ネットワーク、4 つの Intel 内蔵アクセラレータが利用可能です。

🔑 主要機能

💻
カスタム Sapphire Rapids プロセッサ
AWS 専用のカスタム第 4 世代 Intel Xeon Scalable プロセッサを搭載。オールコアターボ 3.2 GHz、最大ターボ 3.8 GHz。Intel TME による常時メモリ暗号化もサポート。
📈
豊富なインスタンスサイズ
large から 48xlarge までの 11 サイズに加え、metal-24xl と metal-48xl の 2 つのベアメタルサイズを提供。最大 192 vCPU、768 GiB メモリ。
⚡
4 つの内蔵アクセラレータ
AMX による ML 高速化は全サイズで利用可能。DSA、IAA、QAT はベアメタルサイズで利用可能。専門的なワークロードを CPU からオフロード。
🌐
高性能インターフェース
DDR5 メモリによる高帯域幅。最大 50 Gbps ネットワーク、最大 40 Gbps EBS 帯域幅、最大 128 EBS ボリューム。48xlarge / metal-48xl で EFA サポート。

🔧 設定方法

前提条件
  • AWS アカウントとアフリカ (ケープタウン) リージョンへのアクセス権限
  • M7i インスタンスタイプのサービスクォータ確認
  • 適切な VPC およびサブネット設定
ステップ 1: M7i インスタンスの起動
Bash ケープタウンリージョンで M7i インスタンスを起動 
# AWS CLI を使用して M7i インスタンスを起動
aws ec2 run-instances \
  --image-id ami-xxxxxxxxxxxxxxxxx \
  --instance-type m7i.4xlarge \
  --region af-south-1 \
  --subnet-id subnet-xxxxxxxxxxxxxxxxx \
  --security-group-ids sg-xxxxxxxxxxxxxxxxx \
  --key-name my-key-pair

--region af-south-1 を指定してアフリカ (ケープタウン) リージョンにインスタンスを起動します。

ステップ 2: 利用可能なインスタンスタイプの確認
Bash 利用可能な M7i インスタンスタイプを確認 
# 利用可能な M7i インスタンスタイプを確認
aws ec2 describe-instance-types \
  --filters "Name=instance-type,Values=m7i.*" \
  --region af-south-1 \
  --query "InstanceTypes[].{Type:InstanceType,vCPU:VCpuInfo.DefaultVCpus,Memory:MemoryInfo.SizeInMiB}" \
  --output table

ケープタウンリージョンで利用可能な M7i インスタンスタイプとスペックを一覧表示するコマンドです。

🎯 ユースケース

🎮 ユースケース 1: ゲームサーバー
アフリカ地域のプレイヤーに低レイテンシーのゲーム体験を提供するために、ゲームサーバーをケープタウンリージョンで実行。カスタム Intel Xeon Scalable プロセッサによる高いシングルスレッドパフォーマンスを活用。
✔ 推奨サイズ: m7i.8xlarge -- ケープタウンリージョンへの近接性により、アフリカ地域のプレイヤーに低レイテンシーの体験を提供
🤖 ユースケース 2: CPU ベース機械学習
AMX アクセラレータを活用して、CPU ベースの機械学習推論を実行。データをアフリカリージョンに保持しながら ML ワークロードを実行可能。
✔ 推奨サイズ: m7i.24xlarge -- AMX アクセラレータにより行列乗算操作が高速化され、ML 推論パフォーマンスが向上
🎥 ユースケース 3: 動画ストリーミングとエンコーディング
アフリカ地域の視聴者向けに、動画のトランスコーディングとストリーミング配信をケープタウンリージョンで実行。M6i と比較して最大 15% のパフォーマンス向上を活用。
✔ 推奨サイズ: m7i.16xlarge -- トランスコーディング処理が高速化され、視聴者へのコンテンツ配信が迅速化

💪 メリット

💰 ビジネス面
  • リージョン拡大: アフリカ (ケープタウン) リージョンで最新世代の Intel ベース汎用インスタンスにアクセス可能
  • コスト効率の向上: M6i 比で最大 15% 優れた価格パフォーマンスにより、汎用ワークロードのコストを削減
  • 柔軟なサイジング: 11 サイズ + 2 ベアメタルにより、ワークロードに最適なインスタンスを選択可能
⚡ 技術面
  • AWS 専用プロセッサ: 他クラウドプロバイダーでは利用できないカスタム Intel Xeon Scalable プロセッサ
  • 内蔵アクセラレータ: AMX による CPU ベース ML の高速化、ベアメタルでの DSA, IAA, QAT 利用
  • 高ネットワーク帯域幅: 最大 50 Gbps ネットワーク、40 Gbps EBS 帯域幅
  • AWS Nitro System: 最新の Nitro System によりセキュリティとパフォーマンスを最適化

⚠️ 制約事項・注意点

⚠ 制限事項
  • DSA, IAA, QAT アクセラレータはベアメタルサイズ (metal-24xl, metal-48xl) でのみ利用可能
  • EFA ネットワーキングは 48xlarge および metal-48xl サイズでのみサポート
  • 新規リージョンでの初期のサービスクォータが制限される場合がある
ℹ 考慮すべき点
  • M7i-flex インスタンスがこのリージョンで利用可能かは別途確認が必要
  • リソースを完全に活用しないワークロードの場合は、M7i-flex の方がコスト効率が良い場合がある
  • 既存インスタンスからの移行時には、パフォーマンステストを実施して期待される改善を確認することを推奨

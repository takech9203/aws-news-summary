Amazon Bedrock

📂 OpenAI 互換 Projects API の提供開始

Mantle 推論エンジンでプロジェクト単位のワークロード分離、アクセス制御、コスト追跡を実現
📅 2026 年 2 月 26 日

📋 概要

Amazon Bedrock が Mantle 推論エンジンで OpenAI 互換の Projects API をサポートしました。Projects API を使用することで、複数のアプリケーション、環境、チームごとに個別のプロジェクトを作成し、ワークロード間の分離を実現できます。

各プロジェクトに IAM ベースのアクセス制御を割り当て、AWS タグを追加することでコストの可視性を向上させることが可能です。Projects API の利用に追加料金は発生しません。

❌ アップデート前の課題

  • 複数のアプリケーションやチーム間でワークロードを論理的に分離する手段がなかった
  • コスト追跡はアカウントレベルで、プロジェクト/チーム単位での正確なコスト配分が困難
  • アクセス制御には個別 AWS アカウントや複雑な IAM ポリシーが必要
  • マルチテナント環境でのガバナンスが課題

✅ アップデート後の改善

  • 単一 AWS アカウント内で複数の AI ワークロードを論理的に分離可能
  • AWS タグ + Cost Explorer でプロジェクト単位のコスト追跡を実現
  • IAM ポリシーでプロジェクトをリソースとして指定し、きめ細かなアクセス制御
  • Responses API / Chat Completions API の両方で利用可能

複数のチームがそれぞれ専用のプロジェクトを通じて Amazon Bedrock の Mantle 推論エンジンにアクセスします。各プロジェクトは IAM によるアクセス制御とタグベースのコスト追跡により分離され、Responses API と Chat Completions API を通じて基盤モデルを利用します。

🔧 主要機能

📁 プロジェクトによるワークロード分離
  • 各プロジェクトは論理的な境界として機能し、アプリケーション、環境、実験を分離
  • 各 AWS アカウントにはデフォルトプロジェクトが自動作成
  • 追加プロジェクトは Projects API を通じて作成可能
🔑 IAM ベースのアクセス制御
  • プロジェクトを IAM ポリシーのリソースとして指定
  • プロジェクト ARN で特定のユーザー/ロールにアクセスを制限
  • 個別 AWS アカウント不要で運用の複雑さを軽減
🏷️ タグベースのコスト追跡
  • プロジェクトに AWS タグを付与し、Cost Explorer でコスト配分を実現
  • ビジネスユニット、チーム、環境などの分類軸でタグ付け可能
  • すべてのレスポンスにタグ情報が含まれる
🔌 OpenAI 互換 API との統合
  • Responses API と Chat Completions API の両方で利用可能
  • OpenAI SDK のプロジェクト ID パラメータを使用
  • cURL では OpenAI-Project ヘッダーで指定

⚙️ 技術仕様 - Projects API と Inference Profiles の比較

🚀 設定方法

前提条件

1. Amazon Bedrock へのアクセスが有効な AWS アカウント
2. IAM 権限 (Bedrock プロジェクトの作成/管理権限)
3. Python 3.7 以降 + OpenAI Python SDK (pip install openai boto3)
4. Amazon Bedrock の API キー

手順

1
環境変数の設定
Amazon Bedrock の API キーとエンドポイント URL を設定します。
2
プロジェクトの作成
Projects API を使用して名前とタグを指定し、新しいプロジェクトを作成します。
3
プロジェクトを使用した推論リクエスト
OpenAI SDK でプロジェクト ID を指定してクライアントを作成し、推論リクエストを送信します。
4
プロジェクトの確認
作成したプロジェクトが正しく設定されていることを確認します。

💻 コードサンプル

Bash 環境変数の設定 
# Amazon Bedrock の認証情報を設定
export OPENAI_API_KEY="<your-bedrock-key>"
export OPENAI_BASE_URL="https://bedrock-mantle.<your-region>.api.aws/v1"
Bash プロジェクトの作成 
# プロジェクトの作成
curl -X POST $OPENAI_BASE_URL/organization/projects \
  -H "Authorization: Bearer $OPENAI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "CustomerChatbot-Production",
    "tags": {
      "Project": "CustomerChatbot",
      "Environment": "Production",
      "Owner": "TeamAlpha",
      "CostCenter": "21524"
    }
  }'
JSON プロジェクト作成レスポンス例 
{
  "arn": "arn:aws:bedrock-mantle:ap-northeast-1:123456789012:project/proj_5d5ykleja6cwpirysbb7",
  "created_at": 1772135628,
  "id": "proj_5d5ykleja6cwpirysbb7",
  "name": "CustomerChatbot-Production",
  "object": "organization.project",
  "status": "active",
  "tags": {
    "Owner": "TeamAlpha",
    "Project": "CustomerChatbot",
    "Environment": "Production",
    "CostCenter": "21524"
  }
}
Python プロジェクトを使用した推論リクエスト 
from openai import OpenAI

# プロジェクト ID を指定してクライアントを作成
client = OpenAI(project="proj_5d5ykleja6cwpirysbb7")

# Responses API を使用した推論リクエスト
response = client.responses.create(
    model="openai.gpt-oss-120b",
    input="Amazon Bedrock の Projects API の利点を説明してください"
)

print(response)

🎯 ユースケース

🏢 マルチテナント SaaS
SaaS プロバイダーが複数のテナントに AI 機能を提供。テナントごとにプロジェクトを作成し、コスト追跡とアクセス分離を実現。
✅ テナント単位のコスト配分とガバナンスの強化
🔄 開発ライフサイクル環境の分離
dev / staging / production の各環境で異なるプロジェクトを使用。環境ごとのアクセス制御とコスト追跡を実現。
✅ 本番データへの不正アクセスリスク排除と環境単位のコスト追跡
💰 ビジネスユニット別コスト管理
複数のビジネスユニットが共通 AWS アカウントで Bedrock を使用。部門ごとにプロジェクトを作成し、コスト配分を実現。
✅ Cost Explorer でタグベースの部門別 AI 支出レポートを生成

✅ メリット

💼 ビジネス面

  • コスト可視性の向上: プロジェクト単位で AI 支出を正確に把握
  • ガバナンスの強化: IAM ベースのアクセス制御でセキュリティポリシーに準拠
  • 運用の簡素化: 個別 AWS アカウント不要、数秒でセットアップ
  • 追加コストなし: モデル推論の使用量に対してのみ課金

🔬 技術面

  • OpenAI SDK 互換: 既存コードベースを最小限の変更で移行可能
  • 柔軟な認証: API キーと AWS 認証情報の両方をサポート
  • マルチ API 対応: Responses API / Chat Completions API の両方で利用可能
  • タグの自動伝播: レスポンスにタグ情報が含まれ、監査が容易

⚠️ デメリット・制約事項

制限事項

  • Mantle 推論エンジン (bedrock-mantle エンドポイント) でのみ利用可能。bedrock-runtime では Inference Profiles を使用
  • 長期 API キーの場合、デフォルトではプロジェクトの取得/一覧のみ許可。作成/更新には追加 IAM ポリシーが必要
  • プロジェクト管理 API は現時点で cURL のみサポート。SDK でのプロジェクト管理は制限あり
  • OpenAI API 仕様の geography パラメータは Amazon Bedrock では無視される

考慮すべき点

  • Mantle エンドポイントが利用可能なリージョンが限定されているため、事前に確認が必要
  • セキュリティ要件が高い場合は、長期 API キーではなく短期キーの使用を推奨
  • プロジェクトのタグ設計は、コスト配分とガバナンスの要件を事前に検討して決定

📝 まとめ

Amazon Bedrock の Projects API により、OpenAI 互換 API を使用するお客様は、単一の AWS アカウント内で複数の AI ワークロードを論理的に分離し、プロジェクト単位でのアクセス制御とコスト追跡を実現できるようになりました。

Projects API の利用に追加料金は発生せず、既存の OpenAI SDK コードベースとの互換性も維持されます。複数のアプリケーション、チーム、環境で Amazon Bedrock を活用している組織は、Projects API を導入してガバナンスとコスト管理を強化することをお勧めします。

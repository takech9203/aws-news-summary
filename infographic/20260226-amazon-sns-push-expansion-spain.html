📨 概要

Amazon Simple Notification Service (Amazon SNS) が、Europe (Spain) リージョンでモバイルプッシュ通知の送信をサポートしました。この拡張により、Amazon SNS はプッシュ通知を 25 リージョンから送信できるようになりました。

Europe (Spain) リージョンのお客様は、ADM、APNs、Baidu、FCM、MPNS、WNS の 6 つのプッシュ通知サービスを利用してモバイルデバイスにプッシュ通知を送信できます。

🔄 Before / After

❌ Before
  • Europe (Spain) リージョンではプッシュ通知機能が利用不可
  • 他のヨーロッパリージョンを使用する必要があった
  • リージョン間のデータ転送が発生しデータローカリティに課題
✅ After
  • Europe (Spain) リージョンから直接プッシュ通知を送信可能
  • 6 つの主要プッシュ通知サービスすべてに対応
  • スペインのデータレジデンシー要件を満たしながら利用可能

⭐ 主要機能

📲 6 つのプッシュ通知サービスをサポート
  • ADM: Amazon Kindle Fire タブレットなどへの通知
  • APNs: iPhone、iPad、Mac への通知
  • Baidu: 中国市場の Android デバイスへの通知
  • FCM: Google Play 搭載 Android デバイスへの通知
  • MPNS: Windows Phone デバイスへの通知
  • WNS: Windows デスクトップおよびタブレットへの通知
⚙️ フルマネージド pub/sub サービス
  • AWS Lambda、Amazon SQS、Data Firehose、HTTP、SMS、E メールなど多様なエンドポイント対応
  • トピックベースのメッセージファンアウトによるスケーラブルなメッセージ配信
  • サーバー管理やインフラストラクチャの運用が不要
🇪🇸 Europe (Spain) リージョンでのフル機能提供
  • プッシュ通知の直接送信とトピック経由のサブスクリプションの両方をサポート
  • デバイストークンの登録、プラットフォームアプリケーションの作成、エンドポイントの管理が可能
  • メッセージアラート、バッジ更新、サウンドアラートとしてモバイルアプリに通知を表示

💻 設定方法

以下の前提条件を満たした上で、AWS CLI を使用してプッシュ通知を設定します。

  • AWS アカウント
  • Europe (Spain) リージョン (eu-south-2) へのアクセス権限
  • 対象プッシュ通知サービスの認証情報 (APNs 証明書、FCM API キーなど)
  • 適切な IAM 権限 (SNS のプラットフォームアプリケーション作成/管理権限)

Step 1: プラットフォームアプリケーションの作成

Bash FCM 用プラットフォームアプリケーション作成 
# FCM 用のプラットフォームアプリケーションを作成
aws sns create-platform-application \
  --name MyAppFCM \
  --platform GCM \
  --attributes PlatformCredential=YOUR_FCM_SERVER_KEY \
  --region eu-south-2

Step 2: プラットフォームエンドポイントの作成

Bash デバイストークンでエンドポイント作成 
# デバイストークンを使用してエンドポイントを作成
aws sns create-platform-endpoint \
  --platform-application-arn arn:aws:sns:eu-south-2:123456789012:app/GCM/MyAppFCM \
  --token DEVICE_TOKEN_FROM_FCM \
  --region eu-south-2

Step 3: プッシュ通知の送信

Bash プッシュ通知送信 
# エンドポイントにプッシュ通知を送信
aws sns publish \
  --target-arn arn:aws:sns:eu-south-2:123456789012:endpoint/GCM/MyAppFCM/xxxx-xxxx \
  --message '{"GCM": "{\"notification\": {\"title\": \"お知らせ\", \"body\": \"新しいメッセージがあります\"}}"}' \
  --message-structure json \
  --region eu-south-2

✅ メリット

💼 ビジネス面
  • スペインおよびヨーロッパのデータ保護規制に準拠しながらプッシュ通知を配信可能
  • Europe (Spain) リージョンから直接送信することで南ヨーロッパのユーザーへのレイテンシーを低減
  • ヨーロッパ内で 7 リージョンから選択可能になり、ビジネス要件に最適なリージョンを選択
🛠️ 技術面
  • 6 つのプッシュ通知サービスで主要モバイルプラットフォームをカバー
  • Lambda、SQS、HTTP など多様なエンドポイントへの統合メッセージ配信
  • フルマネージドで数百万台のデバイスへ自動スケール

⚠️ 制約事項・考慮点

  • プッシュ通知サービスの認証情報は各プラットフォームベンダーから個別に取得する必要がある
  • ペイロードサイズはプラットフォームごとに制限あり (APNs: 4 KB、FCM: 4 KB)
  • デバイストークンの有効期限管理はアプリケーション側で実施する必要がある
  • 既存のプッシュ通知インフラを他のヨーロッパリージョンで運用している場合、移行の必要性を評価すること
  • 認証情報をリージョンごとに設定する必要がある
  • Time to Live (TTL) 設定をプラットフォームごとに適切に構成すること

💡 ユースケース

🛒 スペイン向け E コマースアプリ通知
スペイン市場向けの E コマースアプリから、セール情報や注文ステータスの更新をプッシュ通知で配信。SNS トピックを作成してプッシュ通知を一斉配信。
✔ スペインのデータレジデンシー要件を満たしながら、数万人のユーザーに低レイテンシーでプッシュ通知を配信
📱 マルチプラットフォーム配信
iOS と Android の両方のユーザーに対して、同一メッセージをクロスプラットフォームで配信。message-structure json でプラットフォーム固有のメッセージを一括送信。
✔ 1 回の Publish API 呼び出しで iOS と Android の両プラットフォームに最適化されたプッシュ通知を配信
🚨 リアルタイムアラート配信
ヘルスケアアプリケーションや IoT モニタリングシステムから、緊急アラートをリアルタイムでモバイルデバイスに送信。Lambda 関数からプッシュ通知をトリガー。
✔ Europe (Spain) リージョンから直接配信することで、緊急性の高いアラートのレイテンシーを最小化

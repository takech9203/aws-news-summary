Amazon EC2 I7ie インスタンスが
アフリカ (ケープタウン) リージョンで利用可能に

2026 年 2 月 24 日

☁️ 概要

Amazon EC2 I7ie インスタンスがアフリカ (ケープタウン) リージョンで利用可能になりました。I7ie インスタンスは第 5 世代 Intel Xeon Scalable プロセッサ (Emerald Rapids) を搭載し、オールコアターボ周波数 3.2 GHz で動作するストレージ最適化インスタンスです。

第 3 世代 AWS Nitro SSD を搭載し、クラウドで最高のローカル NVMe ストレージ密度を実現。最大 120 TB のローカル NVMe ストレージを提供し、前世代 I3en と比較して 2 倍の vCPU とメモリを搭載しています。

❌ Before - アップデート前の課題
  • I7ie インスタンスがアフリカ (ケープタウン) リージョンで利用できなかった
  • アフリカリージョンのユーザーは最新世代の高密度ストレージ最適化インスタンスの恩恵を受けられなかった
  • データレジデンシー要件を満たしながら高性能な NVMe ストレージを利用できなかった
✅ After - アップデート後の改善
  • アフリカ (ケープタウン) リージョンで I7ie インスタンスを直接起動可能に
  • 最大 120 TB のローカル NVMe ストレージを活用可能に
  • データローカリティ要件を満たしながらストレージ集約型ワークロードを高性能に実行可能に

I7ie インスタンスは Emerald Rapids プロセッサと第 3 世代 Nitro SSD を搭載し、9 つの仮想サイズとベアメタルサイズで提供されます。

🔑 主要機能

💻
第 5 世代 Intel Xeon Scalable
Emerald Rapids プロセッサをオールコアターボ 3.2 GHz (最大 4.0 GHz) で搭載。Intel TME によるメモリ暗号化をサポートし、I3en 比 2 倍の vCPU とメモリを提供。
💿
第 3 世代 AWS Nitro SSD
最大 120 TB のローカル NVMe ストレージ。I3en 比で最大 65% のリアルタイムストレージ性能向上と 50% の I/O レイテンシー低減。常時 AES-256 暗号化と TWP サポート。
📈
9 つの仮想サイズ
large から 48xlarge まで 9 つの仮想サイズを提供。ベアメタル (metal-24xl、metal-48xl) も利用可能。最大 192 vCPU、1,536 GiB メモリに対応。
🚀
高性能ネットワーキング
最大 100 Gbps のネットワーク帯域幅と最大 60 Gbps の EBS 帯域幅を提供。48xlarge および metal-48xl サイズで EFA をサポート。

🔧 設定方法

前提条件
  • AWS アカウントとアフリカ (ケープタウン) リージョンへのアクセス権限
  • I7ie インスタンスタイプのサービスクォータ確認
  • 適切な VPC およびサブネット設定
ステップ 1: I7ie インスタンスの起動
Bash アフリカ (ケープタウン) で I7ie インスタンスを起動 
# AWS CLI を使用して I7ie インスタンスを起動
aws ec2 run-instances \
  --image-id ami-xxxxxxxxxxxxxxxxx \
  --instance-type i7ie.6xlarge \
  --region af-south-1 \
  --subnet-id subnet-xxxxxxxxxxxxxxxxx \
  --security-group-ids sg-xxxxxxxxxxxxxxxxx \
  --key-name my-key-pair

--region af-south-1 を指定してアフリカ (ケープタウン) リージョンにインスタンスを起動します。

ステップ 2: 利用可能なインスタンスタイプの確認
Bash 利用可能な I7ie インスタンスタイプを確認 
# 利用可能な I7ie インスタンスタイプを確認
aws ec2 describe-instance-types \
  --filters "Name=instance-type,Values=i7ie.*" \
  --region af-south-1 \
  --query "InstanceTypes[].{Type:InstanceType,vCPU:VCpuInfo.DefaultVCpus,Memory:MemoryInfo.SizeInMiB,Storage:InstanceStorageInfo.TotalSizeInGB}" \
  --output table

ケープタウンリージョンで利用可能な I7ie インスタンスタイプとスペックを一覧表示します。

🎯 ユースケース

🗂 ユースケース 1: NoSQL データベース
アフリカ地域のユーザーにサービスを提供するために、MongoDB や Cassandra などの NoSQL データベースをケープタウンリージョンで実行するケースに最適です。
Bash NoSQL データベース向けインスタンスの起動 
aws ec2 run-instances \
  --image-id ami-xxxxxxxxxxxxxxxxx \
  --instance-type i7ie.12xlarge \
  --region af-south-1 \
  --subnet-id subnet-xxxxxxxxxxxxxxxxx \
  --security-group-ids sg-xxxxxxxxxxxxxxxxx
✔ 最大 65% 優れたリアルタイムストレージパフォーマンスと 50% 低い I/O レイテンシーにより、データベースの読み取り/書き込み性能が大幅に向上
📊 ユースケース 2: リアルタイム分析
Spark、Kafka、Splunk などを使用して、大量のデータストリームをリアルタイムで処理し分析する必要があるシナリオに対応します。
Bash リアルタイム分析環境の起動 
aws ec2 run-instances \
  --image-id ami-xxxxxxxxxxxxxxxxx \
  --instance-type i7ie.24xlarge \
  --region af-south-1 \
  --iam-instance-profile Name=RealTime-Analytics-Role
✔ 最大 120 TB のローカル NVMe ストレージと高いコンピューティング性能で、大量データのリアルタイム分析を実行可能
🔍 ユースケース 3: 検索エンジンとインデックス処理
大規模な検索インデックスをローカルストレージに保持し、低レイテンシーで検索結果を返す必要がある場合に最適です。
Bash 検索エンジン向けインスタンスの起動 (EFA 有効) 
aws ec2 run-instances \
  --image-id ami-xxxxxxxxxxxxxxxxx \
  --instance-type i7ie.48xlarge \
  --region af-south-1 \
  --network-interfaces "DeviceIndex=0,SubnetId=subnet-xxxxxxxxxxxxxxxxx,Groups=sg-xxxxxxxxxxxxxxxxx,InterfaceType=efa"
✔ 50% 低い I/O レイテンシーにより、検索クエリのレスポンスタイムが大幅に改善されユーザー体験が向上

💪 メリット

💰 ビジネス面
  • アフリカ (ケープタウン) リージョンのユーザーが最新世代のストレージ最適化インスタンスにアクセス可能
  • I3en と比較して最大 20% 優れた価格パフォーマンスでコストを削減
  • I7i インスタンスと比較して GB あたりの SSD ストレージコストを最大 45% 削減
⚡ 技術面
  • クラウド最高のストレージ密度: 最大 120 TB のローカル NVMe ストレージ
  • I3en 比で最大 65% のリアルタイムストレージパフォーマンス向上
  • 50% 低い I/O レイテンシーで厳格な SLA を持つアプリケーションに最適
  • ベアメタルインスタンスで DSA、IAA、QAT アクセラレータが利用可能

⚠️ 制約事項・注意点

⚠ 制限事項
  • EFA ネットワーキングは 48xlarge および metal-48xl サイズでのみサポート
  • DSA、IAA、QAT アクセラレータはベアメタルサイズでのみ利用可能
  • 新規リージョンでの初期のサービスクォータが制限される場合がある
ℹ 考慮すべき点
  • ローカル NVMe ストレージは一時的であるため、重要なデータは Amazon EBS や Amazon S3 にバックアップが必要
  • I3en からの移行時はアプリケーションの互換性テストを事前に実施することを推奨
  • ストレージ密度とコンピューティング性能の要件に応じて、I7i と I7ie のどちらが適切か評価が必要

💰 料金

I7ie インスタンスはオンデマンドインスタンス、Savings Plans、スポットインスタンスで購入可能です。料金はリージョンとインスタンスサイズによって異なります。

最新の料金については Amazon EC2 料金ページをご確認ください。

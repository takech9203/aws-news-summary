IAM Policy Autopilot - Kiro Power として利用可能に

ワンクリックで IAM ポリシー生成機能を IDE に統合。MCP サーバーの手動設定が不要に

2026 年 2 月 23 日

📋 概要

re:Invent 2025 で発表されたオープンソースの静的コード解析ツールAWS IAM Policy Autopilotが、Kiro Powerとして利用可能になりました。Kiro IDE および Web インターフェースからワンクリックでインストールでき、手動での MCP サーバー設定が不要になります。アプリケーションコードの AWS SDK 呼び出しを決定論的に解析し、アイデンティティベースの IAM ポリシーを生成します。Python、TypeScript、Go をサポートしています。

Before - アップデート前の課題
  • IAM Policy Autopilot を使用するには MCP サーバーの手動設定が必要
  • MCP クライアントの設定ファイルを編集し、コマンドや引数を正しく指定する必要があった
  • セットアップの手間がツール導入の障壁となっていた
After - アップデート後の改善
  • Kiro IDE および Web インターフェースからワンクリックでインストール可能
  • MCP サーバーの手動設定が不要になり、導入障壁が大幅に低下
  • AI アシスタント開発環境にシームレスに統合、生産性が向上

Kiro IDE から IAM Policy Autopilot Power を有効化し、ソースコードの解析からポリシー生成、デプロイまでの流れ

⭐ 主要機能

🔨
ワンクリックインストール
Kiro IDE および Web インターフェースから直接インストール可能。MCP サーバーの手動設定が不要で、設定ファイルの編集やコマンドパスの指定も不要です。
🔎
静的コード解析によるポリシー生成
アプリケーションコードの AWS SDK 呼び出しを決定論的に解析。クロスサービス依存関係を自動検出し、構文的に正しく有効な IAM ポリシーを生成します。
🔓
Access Denied トラブルシューティング
テスト中の Access Denied エラーを検出し、原因を分析してポリシー修正を提案。開発者の承認後にポリシーを更新します。
🌐
マルチ言語サポート
Python、TypeScript、Go のアプリケーションに対応。各言語の AWS SDK 呼び出しを正確に検出し、IAM ポリシーに変換します。

前提条件

  • AWS アカウントと適切な IAM 権限があること

ステップ 2: ポリシー生成の実行

Kiro IDE でチャットセッションを開始し、IAM ポリシーの生成を依頼します。IAM Policy Autopilot がバックグラウンドでソースコードを解析し、必要な IAM ポリシーを生成します。

Prompt Kiro IDE チャット例 
「このアプリケーションに必要な IAM ポリシーを生成してください」

ステップ 3: ポリシーの確認と改善

生成されたポリシーは機能性を優先しているため、最小権限の原則に基づいてスコープダウンすることを推奨します。

Prompt ポリシー改善の依頼例 
「生成された IAM ポリシーを確認して、最小権限に調整してください」

参考: 従来の MCP サーバー設定方法

Kiro 以外の IDE で使用する場合は、従来の MCP サーバー設定が必要です。

JSON MCP サーバー設定例 
{
  "mcpServers": {
    "iam-policy-autopilot": {
      "command": "uvx",
      "args": ["iam-policy-autopilot", "mcp-server"],
      "env": {
        "AWS_PROFILE": "your-profile-name",
        "AWS_REGION": "us-east-1"
      },
      "disabled": false,
      "autoApprove": []
    }
  }
}

👍 メリット

💼 ビジネス面

  • 開発速度の向上: IAM ポリシーの手動作成が不要になり、アプリケーション開発に集中可能
  • 導入障壁の低下: ワンクリックインストールにより、チーム全体での迅速な導入が可能
  • トラブルシューティング時間の短縮: Access Denied エラーの原因を自動分析し、修正を提案

🛠️ 技術面

  • 正確なポリシー生成: 決定論的な静的解析により、AI の幻覚による不正確なポリシーを防止
  • 最新の AWS サービス対応: AWS サービスカタログの更新に追従し、最新の IAM アクションに対応
  • クロスサービス依存関係の自動検出: S3 と KMS など、サービス間の複雑な権限関係を自動的に解決

⚠️ 制約事項・注意点

制限事項
  • 生成されるポリシーはアイデンティティベースポリシーのみ。リソースベースポリシー、Permission Boundaries、SCP、RCP は非対応
  • 機能性を優先したポリシーを生成するため、最小権限の原則に完全に準拠しているわけではない
  • ランタイムで決定される動的な値の解決は非対応
  • サードパーティライブラリ内の AWS SDK 呼び出しの検出は非対応
考慮すべき点
  • 生成されたポリシーはベースラインであり、本番環境デプロイ前にセキュリティ要件に合わせてスコープダウンが必要
  • IAM Access Analyzer などの既存ツールと組み合わせて、継続的に最小権限に改善することが推奨
  • Kiro Power としてのインストールは Kiro IDE に限定。その他の IDE では従来の MCP サーバー設定が必要

ユースケース 1: 新規 AWS プロジェクトの迅速な立ち上げ
新しい Lambda ベースのアプリケーションを開発する際に、S3、DynamoDB、SQS などの複数の AWS サービスを使用する IAM ポリシーを迅速に作成する。
手動で IAM ドキュメントを調査してポリシーを作成する時間を削減し、数分でベースラインポリシーを取得可能
⚡
ユースケース 2: プロトタイプの迅速なデプロイ
ハッカソンや PoC 開発で、IAM ポリシーの作成に時間をかけずにアプリケーションを迅速にデプロイする。
IAM の詳細な知識がなくても、動作するポリシーを迅速に生成し、プロトタイプのデプロイまでの時間を短縮
🔓
ユースケース 3: Access Denied エラーのトラブルシューティング
デプロイ後に Access Denied エラーが発生し、不足している権限を特定して修正する必要がある。
IAM Policy Autopilot がエラーを分析し、不足している権限を特定して修正案を提示。手動でのエラー調査時間を大幅に短縮

💰 料金

無料 - オープンソース

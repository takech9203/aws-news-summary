Amazon EC2 Intel Xeon 6 4 Regions NEW

Amazon EC2 M8i / M8i-flex
4 つの追加リージョンで利用可能に

2026 年 2 月 26 日

概要

Amazon EC2 M8i および M8i-flex インスタンスが US West (N. California)、Europe (Paris)、Asia Pacific (Hyderabad)、South America (Sao Paulo) の 4 つの追加リージョンで利用可能になりました。AWS 専用のカスタム Intel Xeon 6 プロセッサを搭載し、クラウド上の同等の Intel プロセッサの中で最高のパフォーマンスと最速のメモリ帯域幅を提供します。

Before - アップデート前の課題
  • M8i/M8i-flex インスタンスが 4 つのリージョンで利用できなかった
  • これらのリージョンでは前世代の汎用インスタンスを使用する必要があった
  • 最新世代を利用するためにリージョン間データ転送が必要だった
After - アップデート後の改善
  • 4 つの追加リージョンで M8i/M8i-flex インスタンスを直接起動可能に
  • データローカリティ要件を維持しながら最新世代のパフォーマンスを享受
  • より多くのリージョンでワークロードの価格パフォーマンス最適化が可能に

主要機能

📦 M8i-flex インスタンス
汎用ワークロードの大部分で価格パフォーマンスの恩恵を受けられる最も簡単な方法。M8i と比較して 5% 低い価格で 5% 優れた価格パフォーマンスを提供。
  • large から 16xlarge まで最も一般的なサイズを提供
  • 95% の時間でフル CPU パフォーマンスに到達可能
  • ウェブサーバー、マイクロサービスに最適
  • 小中規模データストア、仮想デスクトップ
  • エンタープライズアプリケーション
⚙ M8i インスタンス
すべての汎用ワークロードに最適。特に最大のインスタンスサイズや継続的な高 CPU 使用率を必要とするワークロード向け。
  • 2 つのベアメタルを含む 13 サイズを提供
  • 最大規模向けの新しい 96xlarge サイズ
  • SAP 認定取得済み
  • ミッションクリティカルなワークロードに対応
  • Dedicated Instances / Dedicated Hosts 対応
💻 カスタム Intel Xeon 6 プロセッサ
  • AWS 専用プロセッサ、オールコアターボ 3.9 GHz
  • DDR5 7200MT/s DIMM で 2.5 倍メモリ帯域幅
  • 前世代比 4.6 倍の L3 キャッシュ
  • 常時メモリ暗号化をサポート
🚀 第 6 世代 AWS Nitro Cards
  • 最大 2 倍のネットワークおよび EBS 帯域幅
  • IBC 機能でネットワーク/EBS 帯域幅を 25% ブースト
  • 小パケット処理のスループット改善

設定方法

前提条件
  • AWS アカウント
  • 対象リージョンへのアクセス権限 (us-west-1, eu-west-3, ap-south-2, sa-east-1)
  • 適切な IAM 権限 (EC2 インスタンス起動権限)
  • M8i/M8i-flex インスタンスタイプのサービスクォータ確認
ステップ 1: インスタンスの起動
Bash Europe (Paris) リージョンで M8i-flex インスタンスを起動 
aws ec2 run-instances \
  --image-id ami-xxxxxxxxxxxxxxxxx \
  --instance-type m8i-flex.xlarge \
  --region eu-west-3 \
  --subnet-id subnet-xxxxxxxxxxxxxxxxx \
  --security-group-ids sg-xxxxxxxxxxxxxxxxx \
  --key-name my-key-pair
ステップ 2: 利用可能なインスタンスサイズを確認
Bash 利用可能な M8i インスタンスタイプを確認 
aws ec2 describe-instance-types \
  --filters "Name=instance-type,Values=m8i*" \
  --region eu-west-3 \
  --query "InstanceTypes[].{Type:InstanceType,vCPU:VCpuInfo.DefaultVCpus,Memory:MemoryInfo.SizeInMiB}" \
  --output table

--region パラメータを変更することで、他の新規対応リージョン (us-west-1, ap-south-2, sa-east-1) でも確認・起動が可能です。

ユースケース

🌐 南米向けウェブアプリケーションサーバー
South America (Sao Paulo) で NGINX ベースのウェブアプリケーションを稼働
M7i-flex と比較して最大 60% の NGINX パフォーマンス向上により、南米のエンドユーザーへのレイテンシーを大幅に低減しながら、より多くの同時リクエストを処理可能。
📊 ヨーロッパ向け PostgreSQL データベース
GDPR に準拠しつつ、PostgreSQL データベースを Europe (Paris) で実行
M7i と比較して最大 30% 高速な PostgreSQL パフォーマンスと 2.5 倍のメモリ帯域幅により、データベースクエリの応答時間を大幅に改善。データローカリティ要件も維持。
🤖 アジア太平洋での AI 推論ワークロード
ディープラーニングベースの推奨エンジンを Asia Pacific (Hyderabad) で実行
M7i と比較して最大 40% 高速な AI ディープラーニング推奨モデルの処理と 4.6 倍の L3 キャッシュにより、インドのお客様へのリアルタイム推奨の推論レイテンシーを大幅に短縮。

制約事項・考慮点

⚠ 制限事項
  • 新規リージョンでの初期サービスクォータが制限される場合がある
  • すべてのインスタンスサイズがすべてのリージョンで利用可能とは限らない
  • 既存の M7i/M7i-flex インスタンスからの自動移行はサポートされていない
ℹ 考慮すべき点
  • M8i-flex はコンピューティングリソースを完全に活用しないアプリケーションに最適 (95% の時間でフル CPU 到達可能)
  • 継続的な高 CPU 使用率や最大サイズが必要な場合は M8i を選択
  • 既存の Savings Plans やリザーブドインスタンスの適用範囲を確認
  • ワークロードの特性に応じて M8i-flex と M8i を適切に選択する

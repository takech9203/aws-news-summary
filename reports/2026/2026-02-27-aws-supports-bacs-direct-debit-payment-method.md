# AWS Billing - 英国のお客様向け Bacs Direct Debit 支払い方法のサポート

**リリース日**: 2026 年 2 月 27 日
**サービス**: AWS Billing and Cost Management
**機能**: Bacs Direct Debit 支払い方法

📊 [このアップデートのインフォグラフィックを見る](https://takech9203.github.io/aws-news-summary/20260227-aws-supports-bacs-direct-debit-payment-method.html)

## 概要

AWS は、英国在住のお客様向けに Bacs Direct Debit を新しい支払い方法としてサポートしました。この機能により、GBP ベースの銀行口座から AWS サービスの利用料金を直接かつ自動的に支払うことが可能になります。Bacs 規格をサポートする個人口座またはビジネス口座を安全に接続できます。

従来、英国のお客様はクレジットカード、デビットカード、または EUR ベースの銀行口座のみが支払い方法として利用可能でした。今回のアップデートにより、英国で広く利用されている Bacs Direct Debit による支払いが追加され、クラウド支出の管理がより便利になります。

**アップデート前の課題**

- 英国のお客様はクレジットカード、デビットカード、または EUR ベースの銀行口座のみが利用可能だった
- GBP ベースの銀行口座からの直接引き落としができなかった
- クラウド利用料金の支払いにカード決済を余儀なくされていた

**アップデート後の改善**

- GBP ベースの銀行口座から Bacs Direct Debit による自動支払いが可能
- サインアップ時または既存アカウントの支払い設定から Bacs Direct Debit を追加可能
- 銀行のモバイルアプリまたはオンラインバンキングで安全に認証し、口座をリンク可能
- 追加費用なしで利用可能

## サービスアップデートの詳細

### 主要機能

1. **Bacs Direct Debit の銀行口座リンク**
   - Bacs 規格をサポートする英国の個人口座またはビジネス口座を接続可能
   - TrueLayer を通じた安全な銀行認証プロセス
   - エンドツーエンドの暗号化接続で情報を保護
   - 銀行のモバイルアプリまたはオンラインバンキング認証情報を使用して所有権を検証

2. **新規サインアップ時の設定**
   - AWS サインアップページから「Bacs Direct Debit」を選択可能
   - 銀行を選択し、銀行の認証フローで本人確認を完了
   - デフォルトの支払い方法として自動的に設定

3. **既存アカウントへの追加**
   - AWS Billing コンソールの Payment Preferences ページから追加可能
   - 「Add payment method」から「Bacs Direct Debit」を選択
   - 同じ銀行選択と認証フローで口座を追加
   - 追加後、将来の請求書の支払い方法として利用可能

## 技術仕様

### 支払いプロセス

| 項目 | 詳細 |
|------|------|
| 対応通貨 | GBP (英ポンド) |
| 対象国 | 英国 |
| 対応口座 | Bacs 規格対応の個人口座・ビジネス口座 |
| 認証プロバイダー | TrueLayer |
| 認証方式 | 銀行のモバイルアプリまたはオンラインバンキング |
| 請求タイミング | 請求書発行後 3 営業日以内に引き落とし開始 |
| 支払い完了 | 引き落とし開始から最大 6 営業日 |
| 追加費用 | なし |

### 銀行から取得される情報

| 情報 | 用途 |
|------|------|
| 口座名義人名 | 口座所有権の確認 |
| 口座番号 | 引き落とし先の識別 |
| マンデート ID | Direct Debit の契約識別 |

### API 変更履歴

今回のアップデートに伴う直接的な API 変更はありません。AWS Billing コンソールからの設定のみとなります。

### Bacs Direct Debit 保証

| 項目 | 詳細 |
|------|------|
| 金額・日付・頻度の変更通知 | 引き落とし 2 営業日前に通知 |
| 誤引き落とし時の返金 | 全額即時返金の権利あり |
| キャンセル | 銀行への連絡でいつでもキャンセル可能 |

## 設定方法

### 前提条件

1. 英国在住であること
2. Bacs 規格をサポートする銀行口座 (GBP) を保有していること
3. 銀行のモバイルアプリまたはオンラインバンキングへのアクセスが可能であること

### 手順

#### ステップ 1: AWS Billing コンソールにアクセス

[AWS Billing and Cost Management コンソール](https://console.aws.amazon.com/costmanagement/) にサインインし、ナビゲーションペインから **Payment preferences** を選択します。

#### ステップ 2: 支払い方法の追加

**Add payment method** を選択し、**Bacs Direct Debit** を選択します。

#### ステップ 3: 銀行の選択

提供されたオプションから利用する銀行を選択します。

#### ステップ 4: 銀行認証

銀行のモバイルアプリまたはオンラインバンキングの認証情報を使用してサインインし、ペニーチャージを確認します。AWS のアカウント認証情報ではなく、銀行口座の認証情報を使用してください。接続は暗号化されており、認証情報は保護されます。

#### ステップ 5: 支払い確認と請求先住所の入力

**Confirm payment** を選択して口座所有権の確認を完了します。請求先住所情報を入力し、**Add payment method** を選択して利用規約に同意し、Direct Debit アカウントを追加します。

## メリット

### ビジネス面

- **支払い管理の簡素化**: GBP ベースの銀行口座から自動的に支払いが行われるため、手動での支払い処理が不要
- **コスト削減**: クレジットカード手数料を回避できる可能性がある
- **キャッシュフロー管理の改善**: 銀行口座からの直接引き落としにより、支払い状況の把握が容易

### 技術面

- **安全な認証**: TrueLayer を通じたエンドツーエンド暗号化による安全な銀行認証
- **AWS による認証情報の非保存**: AWS はオンラインバンキングの認証情報にアクセスまたは保存しない
- **Bacs Direct Debit 保証**: 英国の Direct Debit 保証スキームにより、誤引き落とし時の全額返金が保証

## デメリット・制約事項

### 制限事項

- 英国在住のお客様のみが利用可能
- GBP ベースの銀行口座のみ対応
- 支払い完了まで最大 6 営業日を要する場合がある
- 銀行のモバイルアプリまたはオンラインバンキングへのアクセスが認証に必要

### 考慮すべき点

- 口座所有者本人が銀行認証を完了する必要があるため、経理担当者がアカウントオーナーと異なる場合は IAM エンティティを作成してアクセスを付与する必要がある
- Direct Debit のキャンセルは銀行への連絡と AWS への通知の両方が必要
- 引き落とし金額や日付に変更がある場合は 2 営業日前に通知されるが、通知の確認を怠らないよう注意が必要

## ユースケース

### ユースケース 1: 英国のスタートアップ企業

**シナリオ**: 英国のスタートアップ企業が、法人用 GBP 銀行口座から AWS サービスの利用料金を自動的に支払いたい。クレジットカードの利用限度額による制約を受けずに運用したい。

**実装例**:
1. AWS Billing コンソールの Payment Preferences にアクセス
2. Bacs Direct Debit を支払い方法として追加
3. 法人銀行口座をリンクして自動引き落としを設定

**効果**: クレジットカードの利用限度額に依存せず、銀行口座残高の範囲で AWS サービスを利用できます。月次の請求書に基づいて自動的に引き落としが行われ、支払い管理が簡素化されます。

### ユースケース 2: 大企業の経理プロセスの統一

**シナリオ**: 英国の大企業が社内の経理プロセスを統一しており、すべてのベンダー支払いを銀行口座からの Direct Debit で管理する方針を採用している。

**実装例**:
1. AWS アカウントの Payment Preferences から Bacs Direct Debit を追加
2. 企業の承認済み銀行口座を認証してリンク
3. デフォルトの支払い方法として設定

**効果**: AWS の支払いを既存の経理プロセスに統合し、すべてのベンダー支払いを一貫した方法で管理できます。支払い履歴は銀行口座の明細から一元的に確認可能です。

### ユースケース 3: 個人開発者のコスト管理

**シナリオ**: 英国の個人開発者が、個人プロジェクトの AWS 利用料金を個人の銀行口座から直接支払いたい。クレジットカードを使用せずにクラウドサービスを利用したい。

**実装例**:
1. AWS サインアップ時に「Bacs Direct Debit」を支払い方法として選択
2. 個人の銀行口座を認証してリンク
3. 自動引き落としで月次請求を処理

**効果**: クレジットカードなしで AWS サービスを利用でき、個人の銀行口座から直接支払いが行われるため、利用状況と支出を容易に把握できます。

## 料金

Bacs Direct Debit の利用自体に追加費用は発生しません。

| 項目 | 詳細 |
|------|------|
| Bacs Direct Debit 設定 | 無料 |
| 取引手数料 | なし |
| 月額費用 | なし |

## 利用可能リージョン

Bacs Direct Debit 支払い方法は、英国リージョンのお客様に提供されます。AWS サービスの利用自体は、通常どおりすべてのリージョンで可能です。

英国在住の AWS のお客様は、Bacs Direct Debit を使用して GBP ベースの銀行口座から AWS サービスの利用料金を直接支払えるようになりました。TrueLayer を通じた安全な銀行認証により口座をリンクし、追加費用なしで自動引き落としが可能です。英国のお客様で銀行口座からの直接支払いを希望される場合は、AWS Billing コンソールの [Payment Preferences](https://console.aws.amazon.com/billing/home#/paymentpreferences) ページから Bacs Direct Debit を支払い方法として追加することをお勧めします。
